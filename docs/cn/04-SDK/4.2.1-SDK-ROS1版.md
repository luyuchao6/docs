# 4.2.1 SDK-ROS1 版

## 适用版本及系统说明

SDK-ROS1 是针对高擎机电电机开发的通用控制和通信软件接口包，提供以下核心功能：

- ∙🚀 底层硬件通信协议封装
- ∙🤖 多模态电机控制（位置/速度/力矩/复合控制）
- ∙📊 实时状态监测与反馈
- ∙🔧 硬件配置参数化管理
- ∙🔌 支持 CAN-FD 和 CAN 2.0B 通信协议
- 适用版本：
    - SDK程序：`v3.0.0+`
    - 电机固件：`v3.0.0+`
    - 支持系统：`Linux（推荐 Ubuntu 20.04）`
    - 开发环境：`ROS1`

## SDK 例程说明

### 结构

程序主体在 `livelybot_sdk/src` 文件夹下，我们主要使用文件为 `livelybot_bringup`、`livelybot_description`、`livelybot_serial`。

```text
src/
├── livelybot_bringup/      # 启动与管理脚本、配置、launch文件
│   ├── launch/             # launch文件
│   ├── motor_cfg/          # 机器人模型与参数描述（如参数yaml等）
│   └── src/                # 示例程序
├── livelybot_logger/       # 日志记录与服务
├── livelybot_msg/          # ROS消息定义
├── livelybot_oled/         # OLED显示相关节点
├── livelybot_serial/       # 串口通信与底层硬件SDK（核心控制）
├── serial/                 # 通用串口库
├── yesense_ros/            # IMU等传感器相关驱动
```

### 简单介绍

#### motor_rerun

这个程序的功能是控制电机进行正反转运动并周期性地读取并打印每个电机的状态。

#### motor_set_zero

这个程序的功能是设置电机当前位置为零点并周期性地读取并打印每个电机的状态。

#### motor_move_zero

这个程序的功能是控制电机回到零点位置并周期性地读取并打印每个电机的状态。

#### motor_timeout

这个程序的功能是设置电机的超时时间，控制多个电机并定期发送速度指令，同时接收并打印电机的状态反馈信息（如位置、速度、扭矩）。

#### motor_feedback

这个程序的功能是获取和打印电机的状态反馈信息，并不含任何控制指令。

### 详细解析

#### 电机测试程序 motor_rerun

- 头文件

```cpp
#include "ros/ros.h"
#ifndef RELEASE
#include "robot.h"
#else
#include "livelybot_serial/hardware/robot.h"
#endif
#include <iostream>
#include <thread>
#include <condition_variable>
```

- ROS 初始化与设置

```cpp
ros::init(argc, argv, "test_motor");
ros::NodeHandle n;
ros::Rate r(200);
livelybot_serial::robot rb;
```

`ros::init`：初始化 ROS 节点，节点名为 `"test_motor"`。

`ros::NodeHandle`：创建一个 ROS 节点句柄，用于与 ROS 主系统通信。

`ros::Rate r(200)`：设置循环频率为 200Hz，即每秒执行 200 次。

`livelybot_serial::robot rb`：创建一个 `robot` 对象，控制和管理多个电机。

- 获取电机数量并输出日志信息

```cpp
const int motor_num = rb.Motors.size();
ROS_INFO("ALL Motor: %d\n", motor_num);
ROS_INFO("\033[1;32mSTART\033[0m");
```

`motor_num = rb.Motors.size()`：获取电机的数量。

`ROS_INFO`：输出日志信息。

- 控制电机的主循环

```cpp
int cont = 0;
float angle = 0.2;
while (ros::ok())  // 主循环，保持程序运行直到节点关闭
{
    /////////////////////////send
    for (int i = 0; i < motor_num; i++)  // 遍历所有电机
    {
        if(i < (int)motor_num / 2)
        {
            rb.Motors[i]->pos_vel_MAXtqe(angle, 0.1, 10);  // 对前半部分电机发送控制指令
        }
        else
        {
            rb.Motors[i]->pos_vel_MAXtqe(-angle, 0.1, 10);  // 对后半部分电机发送反向控制指令
        }
    }
    cont++;
    if(cont >= 250)  // 每250次反转一次角度的符号
    {
        cont = 0;
        angle *= -1;
    }
    rb.motor_send_2();  // 向电机发送命令
```

`while (ros::ok())`：ROS 主循环，只要 ROS 节点没有被关闭，程序会持续运行。

`rb.Motors[i]->pos_vel_MAXtqe(angle, 0.1, 10)`：向每个电机发送控制指令，控制电机到达指定的位置 `angle`，速度为 `0.1`，最大扭矩为 `10`。前半部分电机控制正向，后半部分电机控制反向。

`angle *= -1`：每 250 次循环后改变电机目标位置的符号，来实现反转运动。

电机控制主要是依靠这两条指令进行控制

设置控制数据：`rb.Motors[i]->pos_vel_MAXtqe(angle, 0.1, 10); `

发送控制指令：`rb.motor_send_2();`

- 循环控制与等待

```cpp
ros::spinOnce();
r.sleep();
```

`ros::spinOnce()`：在每次循环中处理 ROS 回调队列中的所有回调函数。

`r.sleep()`：使当前线程等待，直到达到设定的循环频率。

- ROS 终止

```cpp
ros::spin();  // 保持节点运行，直到关闭
return 0;
```

`ros::spin()`：进入 ROS 主事件循环，处理消息和回调。程序会一直在这个状态下运行，直到 ROS 节点被终止。

## 控制函数

### Motor.cc

- **注意事项：**
    - 统一控制：每次操作控制同一个 CAN 通道上的所有电机。Ω
    - 同一个 can 通道的电机控制模式一致：
        - 同一 CAN 通道下所有电机只能使用相同控制模式，
        - 在调用不同模式的函数时，切换模式时会清空缓冲区。
    - 批量发送：`motor_send_2()` 一次性发送所有缓冲指令
- **例子：**
    - 例 1：

        控制电机 1、2 为位置模式，设置位置为 1，再控制电机 3 为速度模式，设置速度为 0.1，再使用发送函数`motor_send_2()`，只有电机 3 响应进行控制，电机 1 和电机 2 数据都将被清空。

```cpp
// 电机1、2设置为位置模式
robot.Motors[0].position(1)  // 添加到缓冲区
robot.Motors[1].position(1)  // 添加到缓冲区

// 电机3设置为速度模式（模式切换，将先清楚缓冲区，再写入指令）
robot.Motors[2].velocity(0.1)

# 发送指令（此时只有速度模式指令存在）
robot.motor_send_2()  # 仅电机3会响应，电机1、2的指令已被清除
```

    - 例 2：

        下列控制电机的函数是将控制指令的数据帧存储在数组中，在调用`motor_send_2()`函数时统一发送。

```python
# 向缓冲区写控制指令
robot.Motors[0].position(position)
robot.Motors[1].position(position)
robot.Motors[2].position(position)
robot.Motors[3].position(position)
robot.Motors[4].position(position)

# 发送缓冲区的控制指令
robot.motor_send_2()
```

#### 位置控制模式

**说明：**

1. 电机将以最大速度和最大加速度运动到指定目标位置，并返回电机状态信息。
2. 参数说明：
    - `position`：目标位置，单位：弧度（rad）。

**注意：**

1. 该模式下速度与力矩均为最大，运动过程较为激烈。
2. 瞬时电流可能会飙到 5A～10A，若电源响应不够快，或电源电流限制过小，可能导致电机在瞬间获得的电流不足，从而报错。
3. 此模式适用于对响应速度有极端要求的场合，一般不建议使用，如需进行位置控制，推荐使用**梯形控制**模式。
4. 此函数仅将指令写入缓冲区，不会立即发送，需要调用 `motor_send_2()` 才会生效。

```cpp
motor.position(position);
```

#### 位置+速度+最大力矩控制模式

**说明：**

1. 电机以目标速度运动至指定的目标位置，同时限制最大输出力矩，并返回电机状态信息。
2. 参数解析：

**注意：**

2. 此函数仅将指令写入缓冲区，不会立即发送，需要调用 `motor_send_2()` 才会生效。

```cpp

```

#### 运控模式（现不再用该模式，保留是为了兼容老算法，建议使用运控模式 2）

**说明：**

1. 电机输出力矩的计算公式为：
    - 输出力矩 = （目标位置-当前位置） * kp + （目标速度-当前速度） * kd +torque
2. 参数解析：
    - `position`：目标位置，单位：弧度（rad）。
    - `velocity`：目标速度，单位：弧度/秒（rad/s）。
    - `torque`：前馈力矩，单位：牛·米（N·m）。

**注意：**

1. 需要设置合适的 `kp` 和 `kd`，否则控制效果可能较差。
2. 位置是不断积分得到的，所以在低频控制下会有意外情况。不推荐使用该函数，推荐使用运控模式 2 `pos_vel_tqe_kp_kd2`。
3. 此函数仅将指令写入缓冲区，不会立即发送，需要调用 `motor_send_2()` 才会生效。

```cpp
motor.pos_vel_tqe_kp_kd(position, velocity, torque, kp, kd);
```

#### 运控模式 2（MIT模式）

**说明：**

1. 电机输出力矩的计算公式为：

```c
输出力矩 = 位置偏差 * kp + 速度偏差 * kd +前馈力矩
```

2. 模式用法说明
    1. 当 kp=0，kd=0 时，给定 tqe 即可实现给定扭矩输出。在该情况下，电机会持续输出一个恒定力矩。
    2. 当 kp=0，kd≠0 时，给定 vel 即可实现匀速转动。匀速转动过程中存在速度静差，另外 kd 不宜过大， kd 过大时会引起震荡。
    3. 当 kp≠0，kd=0 时，会引起震荡。即对位置进行控制时，kd 不能赋 0，否则会造成电机震荡，甚至失控。
    4. 当 kp≠0，kd≠0 时，有多种情况，这里下面简单介绍两种情况。
        - 当目标位置 pos 为常量，目标速度 vel 为0时，可实现定点控制，系统会使实际位置趋近于 pos，实际速度趋近于 0，最终静止在目标位置。
        - 当 pos 是随时间变化的连续可导函数时，同时 vel 是 pos 的导数，可实现位置跟踪和速度跟踪，即按照期望速度旋转期望角度。
3. 参数解析：
    - `position`：目标位置，单位：弧度（rad）。
    - `velocity`：目标速度，单位：弧度/秒（rad/s）。
    - `torque`：前馈力矩，单位：牛·米（N·m）。

**注意：**

1. 该函数是真正的运控模式，和`pos_vel_tqe_kp_kd`区别是：
    - `pos_vel_tqe_kp_kd`的位置是不断积分得到的，在低频控制下容易出现意外情况。
    - `pos_vel_tqe_kp_kd2` 则避免了此问题，控制更稳定。
2. 该需要设置合适的 `kp` 和 `kd`，否则控制效果可能较差。
3. 此函数仅将指令写入缓冲区，不会立即发送，需要调用 `motor_send_2()` 才会生效。

```cpp
motor.pos_vel_tqe_kp_kd2(position, velocity, torque, kp, kd);
```

### Robot

#### 停止函数

**说明：**

1. 全部电机进入停止模式，电机三相都悬空，使电机可以自由转动。

**注意：**

1. **不会返回电机状态。**
2. 调用此函数时会立即发送指令，无需额外调用发送函数`motor_send_2()`。
3. 会覆盖缓冲区原有数据。

```cpp
robot.set_stop();
```

#### 刹车函数

**说明：**

1. 将电机所有相短接到地，实现“阻尼刹车”效果。
2. 刹车阻力与电机转速成正相关。

**注意：**

1. **不会返回电机状态。**
2. 刹车依靠电机自身阻尼实现，有外力作用时电机仍可缓慢转动。
3. 调用此函数时会立即发送指令，无需额外调用发送函数`motor_send_2()`。
4. 会覆盖缓冲区原有数据。

```cpp
robot.brake();
```

#### 重启函数

**说明：**

1. 对电机执行软件重启。

**注意：**

1. **不会返回电机状态。**
2. 使用重启指令后，电机大约需要 50ms 才可正常控制，函数内部延时 100ms，
3. 电机是单圈绝对值编码器，重启后读取电机位置是单圈数据，即在 -1.717～1.717 弧度内。
4. 调用此函数时会立即发送指令，无需额外调用发送函数`motor_send_2()`。

```cpp
robot.reset();
```

#### 读取状态函数

**说明：**

1. 发送指令读取电机当前状态，用于在不控制电机情况下读取电机状态信息。

**注意：**

1. 调用此函数时会立即发送指令，无需额外调用发送函数`motor_send_2()`。
2. 调用此函数不会对电机进行控制，仅获取状态信息。

```cpp
robot.send_get_motor_state_cmd();
```

#### 发送函数

**说明：**

1. 将缓冲区中已设置的指令数据发送给电机。

**注意：**

1. 所有在 motor 中的电机控制函数（如位置、速度、力矩等）仅写入缓冲区，不会立即生效，需要调用此函数发送指令才会发送，而在 robot 里的控制函数，内部会自行调用 motor_send_2。

```cpp
robot.motor_send_2();
```

### 电机状态函数

#### 获取电机状态函数

说明：

1. 在缓冲区读取电机的 id、控制模式、错误码、位置、速度及力矩信息。

注意：

1. 模式和错误码获取需要 SDK、通讯板、电机固件版本都在 v4 以上才可以读取这两项数据，否则这两项恒为零。
2. 电机状态信息的解析在子线程中完成，无需关注。

```cpp
motor = *m->get_current_motor_state();
            ROS_INFO("ID:%2d,mode: %2d,fluat: %2X,pos: %8f,vel: %8f,tor: %8f", 
            motor.ID, motor.mode, motor.fault, motor.position, motor.velocity, motor.torque);
```

## 配置

`livelybot_description` 文件夹下是电机参数描述文件，`robot_param`目录下有多个电机配置文件可以选择，根据电机使用情况选择文件，并写在`livelybot_description.launch`的路径上，用于选择对应的配置文件。

### 配置文件介绍

程序中提供了一些参考的配置文件，其中部分是对我们公司产品进行配置的电机文件，具体说明如下：

1. 小PI机器人

小PI为双足机器人，采用两条CAN通道，每通道各配置6个电机。适用配置文件

```xml
robot_pi_12dof_cfg.yaml
```

1. PI+机器人

PI+为小型人形机器人，共使用5条CAN通道。其中每条腿配置6个电机，每条手臂配置5个电机，头部及颈部共2个电机。适用配置文件：

```xml
robot_pi_plus_24dof_cfg.yaml
```

1. 小Hi机器人

小Hi为人形机器人，共配备6条CAN通道。腿部每侧6个电机，手臂每侧5个电机，头颈部2个电机，腰部1个电机。适用配置文件：

```xml
robot_hi_23dof_cfg.yaml (不带头部)
robot_hi_25dof_cfg.yaml
```

1. 开源机械臂

为机械臂配置文件，对应配置文件共设置6个电机参数。适用配置文件：

```xml
arm_6dof_cfg.yaml
```

1. 其他开发场景

如用户基于自身项目进行开发，可根据实际需求对配置文件进行调整，或创建新的配置文件以适应不同应用场景。

### 选择电机模型文件

**注意：该文档使用的SDK版本为v4.6以上，该版本选择电机模型文件方式与v4.6以下版本不同**

1. 在`livelybot_bring`文件下的`launch`文件下的`xxx.launch`中选择使用需要的`yaml`文件。

`.launch`的配置如下：

- `file`：电机配置文件的路径，修改该项的配置文件的名称用于选择配置文件

1. 机器人根节点

2. 软硬件参数说明

    - `canport_error_output_flag: false`：表示通讯板CAN通道报错信息输出功能开关，`true`表示使能，`false`表示不使能。
    - `control_type: 9`：表示机器人的控制模式为 9，即运控模式，此处选项对应`motor`中的`fresh_cmd_int16`函数，若直接调用`motor`的控制函数则与该选项无关。

            - `11`：位置-速度-加速度（梯形控制）模式
            - `12`：运控模式 2

```cpp
// 1.调用函数,此时配置文件中为9，即运控模式
motor.fresh_cmd_int16(...);

robot.motor_send_2();
```

    - `imu_limt_num: 1.0`：表示角度的限制值（幅度制，范围：[0，1.57））。
    - `motor_timeout_ms: 0`：表示设置超时时间：

        - 默认设置为 0 表示关闭该功能。
    - `board_special_flag: false`：特殊版本通讯板标志，一般不启用。`true`表示使能，`false`表示不使能。
    - `CANboard_num: 1`：表示机器人使用通讯板数量。
3. 通讯板配置信息
    - `CANboard`：表示机器人的通讯板信息，包含 CAN 通道信息和每个 CAN 通道上的电机信息。
    - `No_1_CANboard`：表示第一块通信板的节点。
    - `CANport_num:1`：表示通讯板使用 CAN 通道数量。这里修改成`1`（**这里是1个通道使用的配置，实际情况根据连接配置**）
    - `CANport`：表示 CAN 通道信息，包含 CAN 通道节点和 CAN 通道上的电机信息。
    - `CANport_1`：表示第一个 CAN 通道的节点，只是一个名称，要求从`CANport_1`开始。
    - `serial_id:1`：表示实际使用的 CAN 通道。
    - `motor_num: 1`：表示这个 CAN 通道上的电机个数。这个修改成`1`（**这里是1个电机使用的配置，实际情况根据连接配置**）
4. 电机配置信息
    - `motor_1`：表示这个 CAN 通道上的第 1 个电机信息。
    - `type: "4438_30"`：电机类型，用于电机输出力矩，填错不影响电机转动，但影响电机输出力矩的准确。可选项有：
        - `3536_32`
        - `4538_19`
        - `5046_20`
        - `5047_09`
        - `5047_36`：这是旧版的 5047_36，现已停产，保留是为了兼容旧算法，推荐使用 5047_36_2。
        - `4438_30`
        - `4438_32`
        - `6056_36`
        - `5043_20`
        - `7256_35`
        - `60sg_35`
        - `60bm_35`
        - `5047_36_2`：相当于新版的 5047_36，推荐使用。
        - `General`
    - `id: 1`：电机 ID
    - `name: "L_low_foot"`：电机名称，可自定义。
    - `pos_limit_enable: false`：位置保护使能标识，`true`表示使能，`false`表示不使能。
    - `pos_upper: 5`：位置上限，单位：弧度（rad）。
    - `pos_lower: -5`：位置下限，单位：弧度（rad）。
    - `tor_limit_enable: false`：力矩限制使能标识，`true`表示使能，`false`表示不使能。
    - `tor_upper: 5`：力矩上限，单位：牛·米（N·m）。
    - `tor_lower: -3`：力矩下限，单位：牛·米（N·m）。

    **注意：**

        - 每个 CAN 通道上的电机 ID 必须从 1 开始。
        - 使用时配置文件中的`id` 要与使用电机 ID 一致。
        - `serial_id`要与实际连接的 CAN 通道一致。
        - 电机按照配置文件中的顺序进行排序
            - 优先按`CANport`顺序排列（`CANport_1` → `CANport_2` → ...)
            - 每个`CANport`内按`motor`顺序排列（`motor1` → `motor2` → ...)
            - 例：
                - 使用了`6dof_*.yaml`配置文件，连接了 12 个电机，为了举例方便，只保留 CANport 和 motor 关系，其他详细配置省略。

```yaml
#配置文件

      motor:
        motor1:...
        motor2:...
        motor3:...
        motor4:...
        motor5:...
        motor6:...
    CANport_2:
      motor:
        motor1:...
        motor2:...
        motor3:...
        motor4:...
        motor5:...
        motor6:...
```

                - 对`CANport_1`的`motor1`、`motor2`和`CANport_2` 的`motor2`、`motor3`进行控制电机的操作如下

```cpp

// CANport_2 的6个电机 → 编号6-11

robot.Motors[0].pos_vel_MAXtqe(...);
```
