# 2.3 Motor Debugging Assistant Troubleshooting

1. The host software cannot detect the USB-to-FDCAN adapter
    ![image](../assets/images/2.3-troubleshooting-img-0.png)
    - Problem description: When connecting via USB cable, opening the host software and connecting to the USBCAN device, the USBCAN device cannot be detected. In Device Manager, the USB-FDCAN device is visible under Universal Serial Bus controllers, but no device is recognized under Ports.
        ![image](../assets/images/2.3-troubleshooting-img-1.png)
    - Cause: A system driver issue — the computer's driver has been modified, preventing connection to the USB-to-FDCAN debug board.
2. After successfully connecting to the motor and entering the position mode in the motor debugging interface, clicking "Add Waveform" and sending a position of 5 turns causes the motor to run briefly then power off with an error.
    ![image](../assets/images/2.3-troubleshooting-img-2.png)

    Cause: In this mode, stopping is achieved by applying reverse torque at the current limit for emergency braking. At the moment of stopping, the current changes sharply, and if the power supply cannot respond in time, it will cause a momentary power drop and trigger an error.

    Solution:

        - Reduce the current limit, or connect a capacitor in parallel on the motor power line.
        ![image](../assets/images/2.3-troubleshooting-img-3.png)
        - It is recommended to use trapezoidal control mode instead.
