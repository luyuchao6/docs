# 4.2.1 SDK-ROS1 Version

## Applicable Versions and System Notes

SDK-ROS1 is a general-purpose control and communication software interface package developed for Gaoqing Mechatronics motors, providing the following core features:

- ∙🚀 Low-level hardware communication protocol encapsulation
- ∙🤖 Multi-modal motor control (position / velocity / torque / composite control)
- ∙📊 Real-time status monitoring and feedback
- ∙🔧 Parameterized hardware configuration management
- ∙🔌 Supports CAN-FD and CAN 2.0B communication protocols
- Applicable versions:
    - SDK program: `v3.0.0+`
    - Motor firmware: `v3.0.0+`
    - Supported OS: `Linux (Ubuntu 20.04 recommended)`
    - Development environment: `ROS1`

## SDK Example Description

### Structure

The main program body is located in the `livelybot_sdk/src` folder. The primary files used are `livelybot_bringup`, `livelybot_description`, and `livelybot_serial`.

```text
src/
├── livelybot_bringup/      # Launch and management scripts, configurations, launch files
│   ├── launch/             # Launch files
│   ├── motor_cfg/          # Robot model and parameter descriptions (e.g., parameter yaml files)
│   └── src/                # Example programs
├── livelybot_logger/       # Logging and services
├── livelybot_msg/          # ROS message definitions
├── livelybot_oled/         # OLED display related nodes
├── livelybot_serial/       # Serial communication and low-level hardware SDK (core control)
├── serial/                 # General-purpose serial library
├── yesense_ros/            # IMU and other sensor-related drivers
```

### Dependencies

```bash
──CMake >= 3.0.2
──C++11 compiler
──libserialport (serial communication)
──yaml-cpp (YAML configuration file parsing)
```

### Detailed Analysis

#### Motor Test Program motor_rerun

- Header files

```cpp
#include "ros/ros.h"
#ifndef RELEASE
#include "robot.h"
#else
#include "livelybot_serial/hardware/robot.h"
#endif
#include <iostream>
#include <thread>
#include <condition_variable>
```

- ROS initialization and setup

```cpp
ros::init(argc, argv, "test_motor");
ros::NodeHandle n;
ros::Rate r(200);
livelybot_serial::robot rb;
```

`ros::init`: Initializes the ROS node with the node name `"test_motor"`.

`ros::NodeHandle`: Creates a ROS node handle for communicating with the ROS master.

`ros::Rate r(200)`: Sets the loop frequency to 200 Hz, i.e., executes 200 times per second.

`livelybot_serial::robot rb`: Creates a `robot` object for controlling and managing multiple motors.

- Obtaining the motor count and outputting log information

```cpp
const int motor_num = rb.Motors.size();
ROS_INFO("ALL Motor: %d\n", motor_num);
ROS_INFO("\033[1;32mSTART\033[0m");
```

`motor_num = rb.Motors.size()`: Gets the number of motors.

`ROS_INFO`: Outputs log information.

- Main loop for motor control

```cpp
int cont = 0;
float angle = 0.2;
while (ros::ok())  // Main loop, keeps the program running until the node is shut down
{
    /////////////////////////send
    for (int i = 0; i < motor_num; i++)  // Iterate over all motors
    {
        if(i < (int)motor_num / 2)
        {
            rb.Motors[i]->pos_vel_MAXtqe(angle, 0.1, 10);  // Send control command to the first half of motors
        }
        else
        {
            rb.Motors[i]->pos_vel_MAXtqe(-angle, 0.1, 10);  // Send reverse control command to the second half of motors
        }
    }
    cont++;
    if(cont >= 250)  // Reverse the angle sign every 250 iterations
    {
        cont = 0;
        angle *= -1;
    }
    rb.motor_send_2();  // Send commands to motors
```

`while (ros::ok())`: The ROS main loop; the program continues running as long as the ROS node has not been shut down.

`rb.Motors[i]->pos_vel_MAXtqe(angle, 0.1, 10)`: Sends a control command to each motor, commanding the motor to reach the specified position `angle` at a velocity of `0.1` with a maximum torque of `10`. The first half of the motors are controlled in the forward direction, and the second half in the reverse direction.

`angle *= -1`: After every 250 loop iterations, the sign of the motor target position is reversed to achieve reciprocating motion.

Motor control primarily relies on these two commands:

Set control data: `rb.Motors[i]->pos_vel_MAXtqe(angle, 0.1, 10);`

Send control command: `rb.motor_send_2();`

- Loop control and waiting

```cpp
ros::spinOnce();
r.sleep();
```

`ros::spinOnce()`: Processes all callbacks in the ROS callback queue during each loop iteration.

`r.sleep()`: Suspends the current thread until the set loop frequency interval has elapsed.

- ROS termination

```cpp
ros::spin();  // Keep the node running until shutdown
return 0;
```

`ros::spin()`: Enters the ROS main event loop to handle messages and callbacks. The program remains in this state until the ROS node is terminated.

## Control Functions

### Motor.cc

- **Notes:**
    - Unified control: Each operation controls all motors on the same CAN channel.
    - Motors on the same CAN channel must use the same control mode:
        - All motors on the same CAN channel can only use the same control mode.
        - When calling functions for different modes, switching modes will clear the buffer.
    - Batch sending: `motor_send_2()` sends all buffered commands at once.
- **Examples:**
    - Example 1:

        Control motors 1 and 2 in position mode with position set to 1, then control motor 3 in velocity mode with velocity set to 0.1, then use the send function `motor_send_2()`. Only motor 3 responds, and the data for motors 1 and 2 will be cleared.

```cpp
// Set motors 1 and 2 to position mode
robot.Motors[0].position(1)  // Add to buffer
robot.Motors[1].position(1)  // Add to buffer

// Set motor 3 to velocity mode (mode switch: the buffer is cleared first, then the command is written)
robot.Motors[2].velocity(0.1)

# Send command (at this point only the velocity mode command exists)
robot.motor_send_2()  # Only motor 3 responds; commands for motors 1 and 2 have been cleared
```

    - Example 2:

        The following motor control functions store control command data frames in an array, which are sent together when `motor_send_2()` is called.

```python
# Write control commands to buffer
robot.Motors[0].position(position)
robot.Motors[1].position(position)
robot.Motors[2].position(position)
robot.Motors[3].position(position)
robot.Motors[4].position(position)

# Send buffered control commands
robot.motor_send_2()
```

#### Position Control Mode

**Description:**

1. The motor moves to the specified target position at maximum speed and maximum acceleration, and returns motor status information.
2. Parameter description:
    - `position`: Target position, unit: radians (rad).

**Notes:**

1. In this mode, both speed and torque are at maximum; the motion is relatively aggressive.
2. Instantaneous current may spike to 5A–10A. If the power supply response is not fast enough or the current limit is too low, the motor may not receive sufficient current instantaneously, causing an error.
3. This mode is suitable for scenarios with extreme response speed requirements and is generally not recommended. For position control, the **trapezoidal control** mode is recommended.
4. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.position(position);
```

#### Velocity Control Mode

**Description:**

1. The motor accelerates at maximum acceleration to the specified target velocity, and returns motor status information.
2. Parameter description:
    - `velocity`: Target velocity, unit: radians per second (rad/s).

**Notes:**

1. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.velocity(velocity);
```

#### Torque Control Mode

**Description:**

1. The motor rotates according to the set target torque, and returns motor status information.
2. Parameter description:
    - `torque`: Target torque, unit: Newton-meters (N·m).

**Notes:**

1. If the set torque is too small, the motor may not be able to rotate.
2. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.torque(torque);
```

#### Voltage Control Mode

**Description:**

1. Controls motor operation via the given Q-phase voltage, and returns motor status information.
2. Parameter description:
    - `voltage`: Q-phase voltage, unit: volts (V).

**Notes:**

1. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.voltage(voltage);
```

#### Current Control Mode

**Description:**

1. Controls motor operation via the given Q-phase current, and returns motor status information.
2. Parameter description:
    - `current`: Q-phase current, unit: amperes (A).

**Notes:**

1. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.current(current);
```

#### Position + Velocity + Maximum Torque Control Mode

**Description:**

1. The motor moves to the specified target position at the target velocity while limiting the maximum output torque, and returns motor status information.
2. Parameter description:
    - `position`: Target position, unit: radians (rad).
    - `velocity`: Target velocity, unit: radians per second (rad/s).
    - `torque`: Maximum allowable output torque, unit: Newton-meters (N·m).

**Notes:**

1. This mode limits the output torque; if the set maximum torque is too small, the motor may not reach the target velocity.
2. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.pos_vel_MAXtqe(position, velocity, torque);
```

#### Motion Control Mode (No Longer Used; Retained for Compatibility with Legacy Algorithms. Use Motion Control Mode 2 Instead.)

**Description:**

1. The motor output torque is calculated as:
    - Output torque = (target position - current position) * kp + (target velocity - current velocity) * kd + torque
2. Parameter description:
    - `position`: Target position, unit: radians (rad).
    - `velocity`: Target velocity, unit: radians per second (rad/s).
    - `torque`: Feed-forward torque, unit: Newton-meters (N·m).

**Notes:**

1. Appropriate `kp` and `kd` values must be set; otherwise the control effect may be poor.
2. The position is obtained by continuous integration, which may cause unexpected behavior at low control frequencies. This function is not recommended; use Motion Control Mode 2 `pos_vel_tqe_kp_kd2` instead.
3. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.pos_vel_tqe_kp_kd(position, velocity, torque, kp, kd);
```

#### Motion Control Mode 2 (MIT Mode)

**Description:**

1. The motor output torque is calculated as:

```c
Output torque = position error * kp + velocity error * kd + feed-forward torque
```

2. Mode usage description:
    1. When kp=0 and kd=0, providing tqe achieves a given torque output. In this case, the motor continuously outputs a constant torque.
    2. When kp=0 and kd≠0, providing vel achieves constant-speed rotation. There is a steady-state velocity error during constant-speed rotation; also, kd should not be too large, as an excessively large kd will cause oscillation.
    3. When kp≠0 and kd=0, oscillation will occur. That is, when controlling position, kd must not be set to 0; otherwise the motor will oscillate or even become uncontrollable.
    4. When kp≠0 and kd≠0, there are various cases; two common cases are briefly introduced below:
        - When the target position pos is constant and the target velocity vel is 0, fixed-point control can be achieved: the system drives the actual position toward pos and the actual velocity toward 0, ultimately coming to rest at the target position.
        - When pos is a continuously differentiable function of time and vel is the derivative of pos, both position tracking and velocity tracking can be achieved, i.e., rotating the desired angle at the desired velocity.
3. Parameter description:
    - `position`: Target position, unit: radians (rad).
    - `velocity`: Target velocity, unit: radians per second (rad/s).
    - `torque`: Feed-forward torque, unit: Newton-meters (N·m).

**Notes:**

1. This function is the true motion control mode. The difference from `pos_vel_tqe_kp_kd` is:
    - `pos_vel_tqe_kp_kd` derives position by continuous integration, which is prone to unexpected behavior at low control frequencies.
    - `pos_vel_tqe_kp_kd2` avoids this problem and provides more stable control.
2. Appropriate `kp` and `kd` values must be set; otherwise the control effect may be poor.
3. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.pos_vel_tqe_kp_kd2(position, velocity, torque, kp, kd);
```

#### Position + Velocity + PD Control Mode

**Description:**

1. The motor output torque is calculated as:
    - Output torque = (target position - current position) * kp + (target velocity - current velocity) * kd
2. Parameter description:
    - `position`: Target position, unit: radians (rad).
    - `velocity`: Target velocity, unit: radians per second (rad/s).

**Notes:**

1. Appropriate `kp` and `kd` values must be set; otherwise the control effect may be poor.
2. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.pos_vel_kp_kd(position, velocity, kp, kd);
```

#### Position + Velocity + Acceleration Control (Trapezoidal Control) Mode

**Description:**

1. The motor moves at a constant acceleration, implementing a trapezoidal velocity profile of accelerate → constant speed → decelerate, while returning status information.
2. Parameter description:
    - `position`: Target position, unit: radians (rad).
    - `velocity`: Target velocity, unit: radians per second (rad/s).
    - `acc`: Acceleration, unit: revolutions per second squared (rps²).

**Notes:**

1. This function only writes the command to the buffer and does not send it immediately; `motor_send_2()` must be called for the command to take effect.

```cpp
motor.pos_vel_acc(position, velocity, acc);
```

### Robot

#### Stop Function

**Description:**

1. All motors enter stop mode; all three phases of each motor are left floating, allowing the motors to rotate freely.

**Notes:**

1. **Motor status is not returned.**
2. Calling this function sends the command immediately; there is no need to additionally call `motor_send_2()`.
3. This will overwrite existing data in the buffer.

```cpp
robot.set_stop();
```

#### Brake Function

**Description:**

1. Shorts all motor phases to ground, implementing a "damping brake" effect.
2. The braking resistance is positively correlated with motor speed.

**Notes:**

1. **Motor status is not returned.**
2. Braking relies on the motor's own damping; the motor can still rotate slowly under external force.
3. Calling this function sends the command immediately; there is no need to additionally call `motor_send_2()`.
4. This will overwrite existing data in the buffer.

```cpp
robot.brake();
```

#### Reset Function

**Description:**

1. Performs a software reset on the motor.

**Notes:**

1. **Motor status is not returned.**
2. After issuing the reset command, the motor requires approximately 50 ms before it can be controlled normally; the function includes an internal delay of 100 ms.
3. The motor uses a single-turn absolute encoder. After a reset, the read motor position is single-turn data, i.e., within -1.717 to 1.717 radians.
4. Calling this function sends the command immediately; there is no need to additionally call `motor_send_2()`.

```cpp
robot.reset();
```

#### Read Status Function

**Description:**

1. Sends a command to read the current motor status, used to read motor status information without controlling the motor.

**Notes:**

1. Calling this function sends the command immediately; there is no need to additionally call `motor_send_2()`.
2. Calling this function does not control the motor; it only retrieves status information.

```cpp
robot.send_get_motor_state_cmd();
```

#### Send Function

**Description:**

1. Sends the command data already set in the buffer to the motors.

**Notes:**

1. All motor control functions in `motor` (such as position, velocity, torque, etc.) only write to the buffer and do not take effect immediately; this function must be called to send the commands. Control functions in `robot` internally call `motor_send_2` on their own.

```cpp
robot.motor_send_2();
```

### Motor Status Functions

#### Get Motor Status Function

Description:

1. Reads the motor's id, control mode, error code, position, velocity, and torque information from the buffer.

Notes:

1. Reading the mode and error code requires the SDK, communication board, and motor firmware to all be at version v4 or above; otherwise these two fields will always be zero.
2. Parsing of motor status information is completed in a sub-thread; no additional attention is required.

```cpp
motor = *m->get_current_motor_state();
            ROS_INFO("ID:%2d,mode: %2d,fluat: %2X,pos: %8f,vel: %8f,tor: %8f",
            motor.ID, motor.mode, motor.fault, motor.position, motor.velocity, motor.torque);
```

    - `control_type: 9`: Indicates that the robot's control mode is 9, i.e., motion control mode. This option corresponds to the `fresh_cmd_int16` function in `motor`; if you call `motor` control functions directly, this option is not relevant.
        - Available options:
            - `1`: Position mode
            - `2`: Velocity mode
            - `3`: Torque mode
            - `4`: Voltage mode
            - `5`: Current mode
            - `6`: Position-velocity-maximum torque mode
            - `7`: Deprecated
            - `8`: Deprecated
            - `9`: Motion control mode
            - `10`: Position-velocity mode with Kp-Kd parameters
            - `11`: Position-velocity-acceleration (trapezoidal control) mode
            - `12`: Motion control mode 2
        - Example:
